# Code Explanation for the NaviFlame EMG Classification Project

## General Explanation

This project implements a sophisticated system for classifying electromyography (EMG) signals into distinct hand gestures, which are then used to control a Unity-based visualizer called "NaviFlame". The core of the project is a machine learning pipeline that uses a **transfer learning** approach, combining a large, pre-trained deep learning model with a smaller, user-specific classifier.

This two-stage design is both powerful and efficient:

1.  **Feature Extraction:** A pre-trained Convolutional Neural Network (CNN), defined in `naviflame/models/transformer.py`, acts as a universal feature extractor. It has been trained on a large dataset to recognize complex patterns in EMG signals. In this project, it is used in a non-trainable (frozen) state to convert raw, filtered EMG data windows into high-level feature vectors (128-dimensional numerical representations).

2.  **Classification:** A much simpler and faster Multi-Layer Perceptron (MLP) model is used as the final classifier. This model takes the feature vectors generated by the CNN and makes the final prediction of which gesture was performed.

The system has two primary modes of operation:

*   **Inference Mode (`naviflame/inference.py`):** This is the real-time mode. It captures live EMG data from a MindRove board, processes it through the feature extractor and the trained MLP classifier, and sends the resulting gesture prediction to the Unity visualizer over a network socket.

*   **Fine-Tuning Mode (`naviflame/record.py` & `naviflame/fine_tune.py`):** This mode allows the system to be adapted to a new user. The user records a fresh set of gestures. The system then uses the powerful CNN to extract features from this new data and trains a **new** MLP classifier specifically on that user's features. This avoids the time-consuming and data-intensive process of retraining the entire deep learning model, making personalization fast and effective.

## Detailed File Explanations

### `naviflame/inference.py`

*   **Purpose:** This is the main script for real-time gesture recognition. It orchestrates the process of data acquisition, processing, classification, and communication with the visualizer.
*   **Key Logic:**
    1.  **Initialization:**
        *   Loads configuration settings from `config.json`.
        *   Connects to the MindRove EMG board.
        *   Loads the three essential model components:
            1.  The Keras CNN feature extractor (`og_fine_tune.h5`).
            2.  The user-specific MLP classifier (`mlp_model.pkl`).
            3.  The user-specific data scaler (`scaler.pkl`).
        *   Initializes digital `BiquadMultiChan` filters using coefficients from the config file to clean the EMG signal.
        *   Sets up a ZeroMQ socket to send predictions to the Unity application.
    2.  **Real-Time Loop:**
        *   The script enters an infinite loop to continuously read data from the board.
        *   It reads both EMG and gyroscope data packets.
        *   The raw 8-channel EMG data is fed through the biquad filters.
        *   The filtered data is passed to the `preprocess_data` utility, which segments it into overlapping windows of 100 samples.
        *   **Classification Trigger:** When a full data window is ready:
            a. The window is fed into the Keras CNN to extract a 128-dimension feature vector.
            b. This vector is normalized by the `StandardScaler`.
            c. The scaled vector is passed to the `MLPClassifier` to get the final gesture prediction (an integer from 0-6).
            d. The predicted gesture is sent to the Unity visualizer.
        *   **Motion Artifact Rejection:** The gyroscope data is used to detect excessive hand motion. If motion exceeds a certain threshold, the system is temporarily paused to prevent inaccurate predictions caused by motion artifacts rather than muscle contractions.

### `naviflame/fine_tune.py`

*   **Purpose:** To train a new user-specific MLP classifier based on newly recorded data. This script embodies the "transfer learning" part of the project.
*   **Key Logic:**
    1.  **Load Data:** It loads the dataset created by `record.py` (e.g., `recorded_gestures.pkl`). This file contains processed EMG data windows and their corresponding gesture labels.
    2.  **Load Feature Extractor:** It loads the pre-trained Keras CNN model (`og_fine_tune.h5`).
    3.  **Feature Extraction:**
        *   It iterates through all the recorded data windows.
        *   For each window, it uses the Keras model's `predict()` method to generate the 128-dimension feature vector from the `dense_8` layer.
        *   It stores these extracted features and their labels in memory.
    4.  **Train the Classifier:**
        *   A new `StandardScaler` is created and `fit` on the extracted features to learn the unique statistical distribution of the new user's data.
        *   A new `MLPClassifier` (with a `(32, 16)` hidden layer architecture) is created.
        *   The `fit()` method is called on the classifier to train it using the scaled features and their corresponding labels.
    5.  **Save Models:** The newly trained and fitted `MLPClassifier` and `StandardScaler` are saved to disk (`mlp_model.pkl` and `scaler.pkl`), overwriting the previous versions. These new models will be immediately used by `inference.py` the next time it runs.

### `naviflame/record.py`

*   **Purpose:** To guide a user through recording the gestures required for the fine-tuning process and to save this data.
*   **Key Logic:**
    1.  **Initialization:** Connects to the MindRove board and initializes the same `BiquadMultiChan` filters used in the inference script to ensure data consistency.
    2.  **Recording Loop:**
        *   It iterates through a list of gestures defined in `config.json`.
        *   For each gesture, it prompts the user to get ready, then provides a countdown before signaling them to start performing and holding the gesture.
        *   During the "hold" period, it records the live, filtered EMG data.
    3.  **Processing and Saving:**
        *   The recorded data for each gesture is processed using the `preprocess_data` function to create the same `(8, 100, 1)` overlapping windows used during inference.
        *   All processed windows from all gestures, along with their correct labels, are collected into a single list.
        *   This final dataset is saved to a pickle file (`recorded_gestures.pkl`) for use by `fine_tune.py`.

### `naviflame/models/transformer.py`

*   **Purpose:** Defines the Python function that constructs the Keras CNN feature extractor model.
*   **Architecture Details:**
    *   Despite the filename, the model is a **Convolutional Neural Network (CNN)**, not a standard Transformer. It is highly customized for time-series signal processing.
    *   **Input:** The model expects data in the shape `(8, 100, 1)`, corresponding to 8 EMG channels, 100 time-steps (samples), and 1 feature per step.
    *   **Custom Blocks:** The architecture is built from custom blocks named `SCNet` and `SFKNet`. These are complex convolutional layers designed to capture features at multiple scales simultaneously. They use techniques like depthwise separable convolutions for computational efficiency.
    *   **Output Layer for Features:** The model's final classification layers are ignored. Instead, the output of the penultimate dense layer, named `dense_8` (which has 128 units), is used as the final feature vector.

### `naviflame/utils.py`

*   **Purpose:** A collection of utility classes and functions used throughout the project to avoid code duplication.
*   **Key Components:**
    *   **`BiquadMultiChan` class:** A crucial component for signal processing. It implements a biquad digital filter that can be applied to multiple channels of data simultaneously. It is initialized with filter coefficients (for low-pass, high-pass, or notch filters) and processes data streams sample-by-sample, maintaining the filter's internal state.
    *   **`preprocess_data` function:** This function takes a stream of filtered EMG data and converts it into the overlapping windows that the CNN model requires. It uses a defined window length (`WIN_LEN=100`) and stride (`STRIDE=10`) to create the segments.
    *   **`MyMagnWarping` and `MyScaling` classes:** These are placeholder custom Keras layers. The original Keras model was likely trained using these custom data augmentation layers. To load the saved model file (`.h5`), Keras needs to know what these layer names refer to. These simple classes are defined so the model can be loaded successfully, but they don't perform any operations during inference; they just pass the data through.

### `config.json`

*   **Purpose:** A configuration file that stores settings and parameters used by the Python scripts. Using a config file allows for easy modification of parameters without changing the source code.
*   **Contents:**
    *   **`"gestures"`:** A list of names for the gestures the system can recognize.
    *   **`"unity_ip"`, `"unity_port"`:** Network details for sending data to the Unity visualizer.
    *   **`"record_time"`, `"wait_time"`:** Durations used during the gesture recording process.
    *   **`"filter_coeffs"`:** The numerical coefficients for the biquad filters, defining their behavior (e.g., cutoff frequency for a low-pass filter).
    *   Other parameters related to the hardware and real-time processing.
